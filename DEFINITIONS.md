# Agent Metric Definitions

## PnL Methods

### Cash Flow PnL
Formula: `yield_earned_usd - gas_paid_usd`

- **yield_earned_usd**: Accrued yield in USD since position opened, based on entry APY applied to
  position size pro-rated by time held.
  `yield_earned_usd = size_usd × (entry_apy / 100) × (days_held / 365)`
- **gas_paid_usd**: Total Solana transaction fees paid (open + any rebalances), converted to USD
  at the SOL price at time of each transaction.
- **Use case**: Measures real cash generated by the agent — what actually landed in the wallet.

### Mark-to-Market PnL
Formula: `current_value_usd - cost_basis_usd`

- **cost_basis_usd**: USD value of capital deployed at time of position open.
- **current_value_usd**: Current USD value of the position (principal + accrued yield at current
  protocol rates).
- **Use case**: Measures total economic value including unrealized gains. More volatile than cash
  flow PnL because it reflects current APY, not the locked-in entry APY.

## Other Metrics

### Capital Utilization
Formula: `sum(size_usd of ACTIVE positions) / total_agent_wallet_balance_usd × 100`

- **Interpretation**: What percentage of the wallet's capital is currently deployed.
- **Phase 1 note**: `total_agent_wallet_balance_usd` is fetched at report time via RPC
  (`getBalance` on the agent wallet address) and converted to USD using a spot SOL price.

### Blended APY
Formula: Weighted average of `entry_apy` across all ACTIVE positions, weighted by `size_usd`.

`blended_apy = Σ(entry_apy_i × size_usd_i) / Σ(size_usd_i)`

- **Use case**: Single number representing the portfolio's current yield rate.
- **Note**: Uses `entry_apy` (locked at open time), not live APY. Live APY drift is tracked
  separately and surfaces as rebalance flags.

### Data Uncertain Flag
Set to `true` when either:

1. Both DefiLlama and the protocol's own API are available, and:
   `abs(apy_defillama - apy_protocol) / apy_protocol > data_uncertainty_threshold_pct`
2. Only one data source is available for a pool (single-source = inherently uncertain).

When `data_uncertain = true`, the scoring engine applies a **risk penalty of 20 points** (weighted
at 0.1 in the formula, so the effective score reduction is 2 points). The flag is also surfaced in
the daily Telegram report.

### Score
Composite 0–100 score computed by the scoring engine:

`score = (apy_score × 0.4) + (liquidity_score × 0.3) + (trust_score × 0.2) - (risk_penalty × 0.1)`

| Component | Scale | Formula |
|-----------|-------|---------|
| apy_score | 0–100 | Linear: 5% APY → 20, 25%+ APY → 100 |
| liquidity_score | 0–100 | Log10 scale: $500k TVL → 20, $50M+ TVL → 100 |
| trust_score | 0–100 | Per-protocol value from `agent_config.yaml` |
| risk_penalty | 0 or 20 | 20 if `data_uncertain`, else 0 |

**Recommendation thresholds:**
- `SUGGEST`: score ≥ 60
- `WATCH`: score ≥ 45
- `SKIP`: score < 45

### Entry APY vs Current APY
- **entry_apy**: The APY recorded at the moment a position was opened. Stored in the `positions`
  table and used for PnL calculations. Does not change after open.
- **current_apy**: The APY fetched from the most recent scanner run for a given pool. Used to
  detect drift and trigger rebalance flags. Not stored persistently — derived from the latest
  `opportunities` row for the pool.
- **APY drop trigger**: If `(entry_apy - current_apy) / entry_apy > rebalance.apy_drop_trigger_pct`,
  the position is flagged for rebalance (transitions to `PENDING_REBALANCE`).
